Interpreter manual/tutorial

So, how to use lympha? I hope this short tutorial for version 1.1 will give all the information needed to use the program.

Requirements

On my Slackware system there are only two additional installations required, namely ruffus and graphviz. The rest of the requirements are pre-installed in the OS. But in case you happen to have another OS, here is a complete list of all the open source requirements:

  • python
  • ruffus
  • bash or an equivalent
  • ghostscript for the ps2pdf command
  • graphviz

Unarchiving

After installing the requirements it's time to unarchive either the zip file or the tar.gz file from the lympha download site. As for example:

tar -xvf lympha-alpha-1.11.tar.gz

Then go into the lympha-alpha-1.11 folder.

cd lympha-alpha-1.11

The content of the folder is shown below.


Data files:
patient_info.pydata about an example patient
factors.pyspecification of determinants, symptoms and other manifestations
procedures.pyspecification of procedures
external_functions.pyexternal functions that to be called from the script(e.g. sending referrals)

Files containing the actual lympha program:
lex.pytranslator from the lympha programming language to python
classes.pyclasses of the program
functions.pyfunctions of the program

Additional files:
LICENSEinformation about the GPL3 license
READMEA little information about the program

Running a script
—Decision support for physicians

First you have to write a script or download one from e.g. the collection of sketch scripts that I've made. In this example I use the script for breast cancer called C50_breast_cancer.lympha. First place the script in same directory as the program files. Then write:

python lex C50_breast_cancer.lympha

This produces a script in python. called C50_breast_cancer.lympha.py. Next write:

python C50_breast_cancer.lympha.py

The computer will now go through the python script and then produce a flowchart called C50_breast_cancer.pdf, that is the same flowchart that you can download from the site where the script is.

Syntax tutorial

The current version supports one script per file. A script, and subsequently a file contains the following parts: head, initiation and node. Further the nodes contains paragraphs.

Head

The head section contains the name of the script:

icd = 'the ICD code'
title = 'the name of the file without the file extension'

For example:

icd = 'C50.-'
title = 'C50_breast_cancer'

Initiation

The initiation section contains descriptions of factors and procedures. An english name must be defined for each factor.

* factors
- variable name ( english name='the english name' )

* procedures
- variable name ( english name='the english name' )

By default all factors are binar. A binar factor is for example a symptom that the patient in question either has or not. Additional arguments can be added. If the factor is a scale, as for example age, the additional arguments would look as follows:

- age ( english name='age' ; real ; unit = 'years old' )

The defenition of procedures is similar to factor defenition, e.g.:

* procedures
- palpation ( english name='palpation' )
- CT ( english name='computed tomography' )

Node

The nodes describe how different factors and procedures are related to each other. Each node has a name. A node can contain set of lists of indications for further action in the pipeline and lists of procedures. Those sets are called paragraphs. Nodes can also contain comments.

node the proceed the node in question>>
name of node:
** comment # the ** stands for the comment
# A list of indications that begins with * all will require all the factors to be true in order to continue the flow of the pipline. The formula in paranthesis describes what interval the factor should be in to be true in the specific case.
* all
- age ( x <= 50 )
# Instead of the * all you can use * any that will require any of the factors to be true.
# A list of procedures begins with * todo as follows:
* todo
- auscultation
